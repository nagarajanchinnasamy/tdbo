Class: tdbo::Database

Generic Database Connectivity Interface that gets implemented by specific Database Connectivity Implementations such as tdbo::SQLite, tdbo::MariaDB, tdbo::PostgreSQL etc. For e.g., if you would like to extend tdbo to support ODBC, then you would write tdbo::ODBC class that inhertis this class and implement the interfaces defined in this class. To extend tdbo, you can use one of the existing implementation classes as a starting point and alter the code specific to your database driver implementation. This is an abstract class and cannot be instantiated directly.

NOTE: As this class is meant for developers of Database Connectivity Implementations, DAO developers can skip this documentation and refer to the documentation of specific database connectivity implementation.

protected method _prepare_condition {conditionlist}

Prepares the condition to be used in WHERE clause of an SQL statement. Implementations mostly invoke this helper method from get method. When supplied with a list of dictionaries where each dictionary containing fieldname-value pairs (For e.g., {{f1 val1 f2 val2} {f1 val3 f3 val4}}) it places an = between fieldname and value and quotes the value (e.g., f1='val1'), then joins the pairs with AND (e.g., f1='val1' AND f2='val2') and then joins the dictionaries with an OR (e.g., ((f1='val1' AND f2='val2') OR (f1='val3' AND f3='val4'))). A value of IS NULL is treated specially by only separating the fieldname and value by space (e.g., f1 IS NULL).

Parameters:
conditionlist
List of dictionaries where each dictionary containing fieldname-value pairs (For e.g., {{f1 val1 f2 val2} {f1 val3 f3 val4}}).
Returns:
The condition to be used in WHERE clause.

Prepares SELECT statement. Implementations invoke this helper method from within get and mget methods.

A dictionary of option-value pairs. Supported options are: -condition, -groupby, -orderby and -fields. Value for -condition option is a string containing the condition to be used in WHERE clause. Implementations invoke _prepare_condition method to form the condition string. Value for -groupby should be a string with valid GROUP BY term. Value for -orderby should be a string with valid ORDER BY term.
Returns:

public method get {schema_name fieldslist conditionlist {format "dict"}}
Pure virtual function / interface to be implemented by derived classes. Retrieves a record, that is uniquely identified by the condition, from a table / view indicated by schema_name. Implementations invoke _prepare_select_stmt helper method to prepare the SELECT statement.
Parameters:
schema_name
Name of the table / view. To be passed to _prepare_select_stmt helper method.
fieldslist
List of field names of the record that needs to be retrieved. To be passed to _prepare_select_stmt helper method as the value for -fields option.
conditionlist
List of dictionaries with each dictionary containing fieldname-value pairs (e.g., {{f1 val1 f2 val2} {f1 val3 f3 val4}}). Implementations invoke _prepare_condition method to prepare the condition that in turn to be passed to _prepare_select_stmt helper method as the value for -condition option.
format
Specifies the format in which the result is to be returned. The value for this option can be either dict or list. Default value is dict. Implementations are expected to format the result accordingly and return as a result of this method.
Returns:
Implementations to return fieldname-value pairs or a list of values as indicated by format argument. If the execution of SELECT statement does not return any record from database, then an empty string should be returned.
public method mget {schema_name args}
Pure virtual function / interface to be implemented by derived classes. Retrieves multiple records from a table / view indicated by schema_name. Implementations invoke _prepare_select_stmt helper method to prepare the SELECT statement.

schema_name
Name of the table / view. To be passed to _prepare_select_stmt helper method.
args
Optional dictionary of option-value pairs. Implementations are expected to support -condition, -groupby, -orderby, -fields and -format options. Implementations can pass these options (except -format) to _prepare_select_stmt method to form the SELECT statement. -format option is expected take any one of dict, list, llist as its value. dict is the default value for -format option.
